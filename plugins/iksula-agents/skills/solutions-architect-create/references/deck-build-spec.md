# Deck Build Spec

Slide-by-slide structure for the two decks. Both decks draw from the SAME source of truth (grounding statement + approved differentiation edge). Build with the `pptx` skill. Keep them consistent: same buyer, same need, same edge — different audience and framing.

---

## Deck A — Internal Solution Introduction Deck
**Audience:** Iksula internal (sales, content, delivery, leadership). **Purpose:** brief the team AND serve as the structured input contract for downstream content-creator and delivery agents.

1. **Title** — solution name, one-line definition, "Internal — Solution Introduction", date/version.
2. **Solution at a glance** — name, one-line definition, what it's built on (Iksula asset/runtime), the value lever in one phrase.
3. **Why this, why now** — the opportunity traced to the vertical mapping (which shortlist row, what tier/score), and the trigger making it timely.
4. **Buyer profile** — economic buyer / user / influencer; for the economic buyer: role, the KPI they own, what keeps them up at night.
5. **The need (job-to-be-done)** — the pain in the buyer's language, tied to the mapped L3 tasks and framed in the archetype's economic terms (e.g. cost-to-serve, leakage).
6. **Competitive landscape** — the matrix (players × offer/buyer/model/claim/strength/gap).
7. **Table-stakes vs gaps** — what everyone promises (don't pitch these) vs the unserved gaps / white space.
8. **How we differentiate** — the approved edge: the wedge, stated plainly.
9. **Our edge in practice** — the Iksula asset/runtime/data/delivery/price that powers it; why it's defensible (right-to-win).
10. **Solution shape** — what's in v1, what's explicitly out, the value metrics we'll claim.
11. **Handoff section (the contract)** — structured facts for downstream agents:
    - Target segments & ICP
    - Scope in / out
    - Value metrics & proof points needed
    - Key messages / positioning statement
    - Open questions & assumptions to resolve
12. **Appendix** — sources for all competitive claims.

> Slides 4–11 are the machine-readable core. Keep them structured (tables/bullets with clear labels), not prose — downstream agents parse this.

## Deck B — Client Pitch Deck
**Audience:** the prospective client (CXO / economic buyer). **Purpose:** sell. Narrative-led, jargon-free, leads with the buyer's KPI. No competitive matrix, no internal handoff section.

**Corporate intro (always open with these three — pull content from existing Iksula decks, never invent):**
1. **Title** — solution name + a benefit-led tagline in the buyer's terms.
2. **About Iksula / Who we are** — the firm in one line + headline stats (years, engineers, clients, scale) + the capability pillars.
3. **Our Services** — the full-spectrum service set, each with a proof metric and named clients.
4. **Roster of clients** — the trusted-by logo wall, grouped by segment. **Reuse the actual client logo images** extracted from the source deck's logo-wall slide (e.g. python-pptx: pull each picture's blob + position, rescale onto the brand canvas) — do NOT substitute a text list of brand names. A wall of real logos is the point.

**Solution story:**
5. **Your world today** — the buyer's reality and the cost of the problem, in their economic terms (numbers if defensible).
6. **What it's costing you** — quantify the pain / the status-quo tax.
7. **A better way** — the future state, before/after.
8. **The solution** — what Iksula delivers, in outcome terms (what changes for them, not feature lists).
9. **Why Iksula / why now** — the edge translated into client value (not internal jargon).
10. **Proof** — headline outcome metrics, the asset/runtime that de-risks delivery.
11. **Client case studies** — 2–3 real, named clients with the challenge and the measurable outcome, sourced from existing Iksula decks (e.g. corporate decks, case-study folder). Never fabricate a client or a number.
12. **Next step** — a concrete, low-friction first step (pilot / assessment / workshop).

> Rule: every slide answers "so what for the buyer's KPI?" If a slide is about Iksula rather than the buyer's outcome, recut it.

> **Brand:** build to the Iksula PPT Brand Spec — Carlito, red `#9A0D15`, light card slides, dark/red panels only for cover/section/closing, real iksula wordmark. Verify colours against the spec; never approximate.

## Consistency checklist (run before delivery)
- Same buyer named in both decks.
- Same need / economic lens in both.
- Same single differentiation edge as the spine of both.
- Every competitive claim in Deck A is sourced in its appendix.
- Deck B contains zero internal jargon and no competitive matrix.
- Deck A's handoff section is complete enough that a downstream agent needs no extra context.
